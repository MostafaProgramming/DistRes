Demo Talking Points

  1. Every browser client maps to an authenticated socket session on the C++ server.
  2. Read locks can be held by several clients at once.
  3. The writer lock is exclusive and waits until active readers stop.
  4. Committed writes publish update events to subscribed clients.
  5. The dashboard is read-only, so it observes coordination without changing resource access.
